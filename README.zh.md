## 📖 项目简介

**NEON-UNO** 是一款受经典 UNO 启发的同场多人游戏体验。玩家使用自己的智能手机作为游戏手柄，与共享的大屏幕（投影仪或大显示器）实时互动。游戏以霓虹灯风格设计，将参与者随机分为两支队伍 —— **霓虹蓝队** 🔵 和 **霓虹粉队** 🟣 —— 双方竞争填满各自的积分条，最终夺得胜利。

---

## ✨ 主要功能

| 功能              | 说明                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------ |
| **大屏幕**     | 展示游戏竞技场、当前牌、玩家分布、积分条以及炫酷的视觉特效。 |
| **手机手柄**   | 玩家只需扫描二维码即可加入，手牌按颜色自动整理，适配横竖屏。                 |
| **双队对抗**   | 队伍随机分配，得分通过复杂的实时权重计算影响各队的共同积分条。           |
| **状态重连**   | 支持断线重连和页面刷新：玩家状态（手牌、分数等）在刷新后依然保留。       |
| **病毒事件**   | 随机病毒事件冻结游戏：摇晃手机治愈自己，否则将受到惩罚！                 |
| **特殊牌效果** | 效果始终针对对方队伍（+2/+4 随机对手摸牌，冻结全队）。                   |
| **触觉反馈**   | 加速度计和振动让体验更加身临其境。                                       |
| **实时反馈**   | 出牌瞬间在投影屏幕上显示得分弹窗、激光射击特效。                     |
| **表情互动**   | 玩家可发送实时表情，游戏结束后进入“表情狂欢”模式。     |

## 🎮 游戏规则

> 本节根据源代码（`server.js`）中的实现逻辑详细说明游戏规则。

### 🃏 牌组构成

游戏使用共 **108 张牌**的牌组：

| 颜色                       | 牌张                                                                                 |
| -------------------------- | ------------------------------------------------------------------------------------ |
| 🔴 红、🔵 蓝、🟢 绿、🟡 黄 | `0`（×1）、`1`–`9`（各 ×2）、`+2`（×2）、`跳过`（×2）、`反转`（×2）—— 每色 **25 张** |
| ⚫ 黑色                    | `百搭`（×4）、`百搭 +4`（×4）                                                        |

### 🚀 游戏准备

1. 主持人在投影仪屏幕上按下 **"开始"** 后，游戏正式启动。
2. 每位玩家初始获得 **7 张手牌**。
3. 翻开一张非黑色牌，正面朝上置于中央，作为**当前活跃牌**。
4. 玩家连接时，**随机分配**至蓝队或粉队。
5. 双方起始积分均为 **50 分**（中央积分条）。

### ⚡ 回合流程 —— "最快者胜！"

> **没有出牌顺序！** 所有人同时出牌，第一个打出有效牌的玩家赢得本轮。

出牌必须满足以下**至少一个**条件：

- ✅ 与当前活跃牌**颜色相同**
- ✅ 与当前活跃牌**数值（或动作类型）相同**
- ✅ 是**黑色牌**（百搭或百搭 +4）—— 随时可出

若手中没有合适的牌，玩家可以**摇晃手机**从牌堆摸一张牌。

### 📊 积分系统

NEON-UNO 采用动态权重评分系统，每次成功出牌后的得分不是固定的，而是根据多重因素计算出的**浮点得分**：

**`最终分 = 基础分 (人类2.0 / AI 按等级) × 层1乘数 (手牌数) × 层2乘数 (颜色多样性) × 层3乘数 (同色连打)`**

> 统一化逻辑：AI 和人类玩家现在共用同一套结算函数（`processPlayCard`），确保计分、罚牌和特殊牌效果的绝对公平一致。

#### 1. 层1：手牌数量乘数 (Hand Count)
手牌越少，得分越高（高风险奖励）。
- **1张手牌**：**1.8x**
- **5-7张**：**1.0x** (基准)
- **10张以上**：快速衰减至约 **0.4x**，最低 **0.3x**。

#### 2. 层2：颜色多样性乘数 (Color Variety)
手牌颜色越单一（说明在控场），得分越高。包含黑色百搭牌在内的5种颜色。
- **1色**：**1.8x**
- **2色**：**1.3x**
- **3色**：**1.0x**
- **4色**：**0.7x**
- **5色**：**0.3x**

#### 3. 层3：同色连打乘数 (Streak)
惩罚无脑连出同一种颜色。
- **换色/第一张**：获得 **1.2x** 奖励。
- **连续第2张同色**：**1.0x**。
- **连续第5张同色**：约 **0.5x**，最低渐近于 **0.35x**。

**获胜条件**：**率先达到 100 分的队伍获胜**。清空手牌不再直接获胜，但能为队伍换取巨大的积分加成，清空后玩家需继续摸牌参与。

### 🔮 特殊牌效果

| 牌型                         | 效果                                                  |
| ---------------------------- | ----------------------------------------------------- |
| **`+2`（Draw 2）**           | 对方队伍中**随机一名对手**摸 **2 张牌**。             |
| **`百搭`（Wild）**           | 出牌玩家选择下一张活跃牌的颜色。                      |
| **`百搭 +4`（Wild Draw 4）** | 玩家选择颜色，**同时**对方随机一名对手摸 **4 张牌**。 |
| **`跳过`（Skip）**           | 对方**整支队伍**被**暂时冻结**（持续数秒）。          |
| **`反转`（Reverse）**        | 对方**整支队伍**被**暂时冻结**（效果同跳过）。        |

> 💡 打出百搭牌后，玩家选择的颜色将成为本轮后续的活跃颜色。

### 🦠 病毒事件

每隔 **1 到 2 分钟**，系统会不定时触发一次**病毒**警报：

1. **5 秒**倒计时开始。
2. 每位玩家必须**摇晃手机**来"治愈"自己。
3. 倒计时结束时：
   - **已治愈**的玩家：无惩罚。
   - **未治愈**的玩家：获得 **+2 张惩罚牌**。
4. 下一次病毒警报随机重新计时。

> ⚠️ 保持警惕！病毒随时可能爆发。

### 📣 UNO 规则

- 当手中**只剩一张牌**时，在出牌前必须按下 **UNO** 按钮。
- 若出了倒数第二张牌却**未喊 UNO**，将自动受到 **+2 张惩罚牌**的处罚。

### ♻️ 牌堆补充

当摸牌堆不足 **10 张**时，系统会自动加入一副全新的洗好的 108 张牌。游戏永远不会因牌不够而中断。

## 🤖 AI 玩家

主持人可在大厅界面底部通过「🤖 添加AI」按钮添加最多 **5 个 AI 玩家**。AI 玩家现在拥有更平衡的计分系统和更复杂的决策逻辑。

#### 🧠 AI 等级说明

| 性格 | 基础分 | 速度特性 | 决策算法 | 行为特征 |
|---|---|---|---|---|
| **⚡ 小白型** (_noob_) | **0.7** | 极快 | **乱打流** —— 优先选择与中央牌**同颜色**的牌。 | 这种出牌方式极易堆积同色惩罚（Streak），以此抵消其极高的出牌频率。 |
| **⚖️ 进阶型** (_casual_) | **1.0** | 中等 | **战术启发式** —— 优先权重：功能牌 > 颜色一致 > 数值一致 > 百搭。 | 决策更具威胁性，尝试在中期控制局面。 |
| **🧠 专家型** (_pro_) | **1.3** | 较慢 | **2步看前算法 (Lookahead)** —— 模拟后续可能局面，最大化“分/秒”产出比。 | 能够精准评估攻击获利，执行悲观抽牌策略（除非极大获利否则不抽牌）。 |

#### ⚙️ AI 系统细节

- **三层动态延迟**：AI 的出牌速度不再是固定值，而是分为三个场景：`同色出牌`（0.8s-1.2s）、`换色出牌`（1.5s-2.0s）、`黑牌/换色思考`（2.5s+）。
- **重新思考逻辑**：当中央牌颜色被改变时，所有已规划的 AI 计时器会重置，模拟 AI 针对新情况重新进行战术计算。
- **状态同步**：AI 现已深度集成 `frozen` 状态机，能够被对方人类或 AI 玩家通过 `Skip` 或 `Reverse` 牌逻辑冻结。

## 📦 安装步骤

1. 克隆仓库并进入项目目录。
2. 安装依赖：
   ```bash
   npm install
   ```
3. 生成自签名 SSL 证书（**必须步骤** —— 移动浏览器需要 HTTPS 才能访问加速度计）：
   ```bash
   node gen-cert.js
   ```
   _此步骤会在项目根目录生成 `cert.pem` 和 `key.pem` 文件。_

## 🚀 启动游戏

### 开发模式（热重载）

在两个终端中并行运行两个服务器：

```bash
# 终端 1 —— WebSocket/Express 服务器
npm run dev:server

# 终端 2 —— Vite 服务器（Vue.js 客户端）
npm run dev:client
```

### 生产模式

```bash
# 1. 构建 Vue.js 应用
npm run build

# 2. 启动服务器
npm start
```

游戏将在 **`https://localhost:3000`** 上运行。本地网络 IP 已嵌入投影仪屏幕上显示的二维码中，其他设备可直接扫码连接。
